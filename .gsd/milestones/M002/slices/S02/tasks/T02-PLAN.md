# T02: Wire Serwist service worker, offline page, and tsconfig update

**Slice:** S02 — PWA Shell & Home Screen
**Milestone:** M002

## Description

Integrate Serwist into the Next.js build pipeline: wrap `next.config.ts` with `withSerwistInit`, create the service worker source at `app/sw.ts`, add a static offline fallback page at `app/~offline/page.tsx`, and update `tsconfig.json` for web worker types. After this task, `npm run build` generates `public/sw.js` with a precache manifest and the offline fallback URL precached.

**Key constraint:** `output: "standalone"` must remain in `next.config.ts` — the Serwist wrapper goes around it, not replacing it.

**Known limitation (D028):** Service workers only register on HTTPS or localhost. On plain HTTP LAN (`http://192.168.x.x:3000`), the SW will not install. The manifest-based standalone install and icons still work.

## Steps

1. **Update `next.config.ts`**: Import `withSerwistInit` from `@serwist/next`. Compute a `revision` string from `git rev-parse HEAD` (with `crypto.randomUUID()` fallback). Create the wrapper: `const withSerwist = withSerwistInit({ swSrc: "app/sw.ts", swDest: "public/sw.js", disable: process.env.NODE_ENV === "development", additionalPrecacheEntries: [{ url: "/~offline", revision }] })`. Export `withSerwist(nextConfig)` where `nextConfig` keeps `output: "standalone"`.

2. **Create `app/sw.ts`**: Use the exact Serwist pattern from the research doc. Triple-slash references (`no-default-lib`, `esnext`, `webworker`). Import `defaultCache` from `@serwist/next/worker`, `Serwist` from `serwist`. Declare `self.__SW_MANIFEST`. Instantiate Serwist with `precacheEntries: self.__SW_MANIFEST`, `skipWaiting: true`, `clientsClaim: true`, `navigationPreload: true`, `runtimeCaching: defaultCache`, `fallbacks.entries` mapping document requests to `/~offline`. Call `serwist.addEventListeners()`.

3. **Create `app/~offline/page.tsx`**: A simple static React component — no data fetching, no client-side JS. Display a centered message: "You're offline — theFridge server is unreachable. Check that the home device is running." Use inline styles with CSS variables (`var(--color-bg)`, `var(--color-text)`, `var(--color-text-muted)`).

4. **Update `tsconfig.json`**: Add `"webworker"` to the `lib` array. The final array should be `["dom", "dom.iterable", "esnext", "webworker"]`.

5. **Verify**: Run `npm run type-check` (must exit 0). Run `npm run build` (must exit 0 and produce `public/sw.js`).
